# Маппинг пиксельной сетки
Работа с датасетом для проекции координат (x, y) двух камер (top/bottom) в координаты (x', y') третьей камеры (door2). 

Рассматривались подходы классической гомографии, TPS, триангуляции Делоне, применение многослойных перцептронов. Последнее является baseline решением.


## Установка
Для начала работы настройте виртуальное окружение Python и установите зависимости из `solution/requirements.txt`, из корневой папки проекта:
``` Python
pip install -r solution/requirements.txt
```


## Использование
Перед использованием необходимо обучить модель на тренировочном датасете. Если до этого модель не обучалась - запустите `solution/train.py`. В папке models должны появиться 2 модели и 4 скейлера, а также метрики на валидационном датасете(`mlp_metrics.json`).

Когда модель обучена, можно использовать файл `predict.py` для получения проекции. Пример использования из корневой папки:
```Python
python solution/predict.py -x 100 -y 100 -s top   
```

Принимаемые аргументы(обязательные):
- -x <значение> (координата)
- -y <значение> (координата)
- -s/--source <top/bottom> - источник координат, камеры bottom или top

Необязательные аргументы:
- -m/--method <mlp/delaunay/homography> - метод для использования predict. Нужно заранее обучить соответствующие модели, delaunay и homography обучаются в `unused/`.
- --no-print_result - не выводить результат преобразования в консоль.

Альтернативно можно вызвать predict из `solution/predict.py` и передать необходимые параметры.


## Визуализация
В процессе разработки появился файл `eval.py`. Пример его использования:
```Python
python solution/eval.py --method 'mlp' --visualize --visualization_limit 3
```

Метод рассчитывает метрики на валидационных данных и сохраняет в папку из которой был вызван в виде текстового файла, а также выводит картинку с door2 и ошибками предсказаний при указании параметра `--visualize`.
